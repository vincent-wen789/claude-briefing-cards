# briefing-card

[English](README.md) · [中文](README.zh-CN.md) · **日本語**

Claude Code 用のプラグイン。チャット内に描画される**意思決定カード / 進捗カード / 確認カード**(クリック可能なボタン付き)と、**T0–T3 の報告粒度ルーター**を提供します。ルーターは毎ターン、ステータスや決定事項をプレーンテキスト・簡易質問・インタラクティブカード・HTML ファイルのどれで返すかを自動で判定します。

## なぜ skill ではなくプラグインなのか

skill の `description` が確実に発火するのは**明示的なトリガー**(ユーザーがカードを求めたとき)だけです。「決定や進捗を報告するときは既定でカードを描く」という**常時発動型の自動トリガー**は担えません。この挙動には、毎ターン文脈に常駐する standing instruction が必要です。素の skill フォルダにはそれが同梱されないため、新しいマシンでは「入れても入れていないのと同じ」に感じられます。

本プラグインは **SessionStart フック**(`hooks/session-start`)でこれを解決します。セッション開始 / clear / compact のたびに、ルーティング規則を常駐コンテキストとして注入します。インストールするだけで動作し、`CLAUDE.md` を編集する必要はありません。

## インストール

Claude Code 内で(シェルではなく):

```
/plugin marketplace add vincent-wen789/claude-briefing-cards
/plugin install briefing-card@vincent-plugins
```

その後、**新しいセッションを開始**してください。フックはセッション開始時に注入されるため、インストールした当のセッションには反映されません。

## 動作確認

- **簡易チェック:** 新しいセッションで「T0–T3 の報告カードに関する常駐指示はありますか?」と尋ねます。ルーティングを復唱できれば、フックは有効です。
- **本チェック:** カードを求めずに、複数行の進捗報告やトレードオフのある 2 択を出させます。プレーンテキストではなく、自動でカードを描く / ルーティングされれば成功です。

## 構成

```
.claude-plugin/plugin.json        # プラグインのマニフェスト
.claude-plugin/marketplace.json   # マーケットプレイス登録(source: "./")
hooks/hooks.json                  # SessionStart → session-start を実行
hooks/session-start               # 常駐ルーティング文脈を注入するスクリプト
hooks/session-context.md          # ルーティング本体(挙動の変更はここ)
skills/briefing-card/             # skill 本体(テンプレート + ルーティング)
```

## 注記

これは**個人版**です。語り口・トリガー語・「既定でカードを出す」という傾向は特定ユーザー向けに調整されており、内容は中国語です。不特定多数に配布するには、語り口を汎用化し、積極的な既定動作をオプトインにする必要があります(さもないとカードを出しすぎます)。
